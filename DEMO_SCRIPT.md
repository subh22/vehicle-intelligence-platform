## SCENE 3 — Architecture Overview (1:15 – 2:00)

**[Screen: Open the README.md on GitHub or draw the diagram]**

> "Here's how it works."

**[Point to each block as you say it:]**

> "A CCTV image of the vehicle enters from the left.
> It goes through two computer vision models —
> one identifies the vehicle type,
> the other detects visible damage like dents and scratches.
>
> At the same time, the customer's text request
> is sent to Claude — Anthropic's large language model —
> which extracts their intent: are they here for a routine service?
> An insurance claim? An emergency?
>
> Vehicle metadata — year, mileage, ownership —
> is pulled from a structured dataset.
>
> All three signals are fused in the pipeline,
> which calculates a service priority
> and outputs a single structured service record — in real time."

## SCENE 4 — Project Structure (2:00 – 2:45)

**[Screen: VS Code or file explorer showing the project folder]**

> "Let me quickly walk through the codebase."

**[Click through folders as you narrate:]**

> "`core/vision/` — our two EfficientNet-B0 models.
> One trained on the Kaggle Vehicle Classification Dataset —
> 5,600 images across 7 vehicle categories.
> The other trained on the Car Damage Detection dataset —
> classifying dents, scratches, shattering, and dislocation.
>
> `core/nlp/` — the Claude Haiku intent extractor.
> A carefully designed system prompt turns raw customer text
> into structured JSON: intent, urgency, key concerns, sentiment.
>
> `core/data/` — the CarDekho metadata processor,
> which enriches every request with vehicle age,
> mileage category, and ownership history.
>
> `core/pipeline.py` — the brain.
> It fuses all signals and applies business rules
> to determine whether a case is low, medium, or high priority.
>
> And `api/` — a production-ready FastAPI server
> with five endpoints, full Swagger documentation,
> and Docker support."

## SCENE 10 — Priority Logic (6:15 – 6:45)

**[Screen: Open core/pipeline.py in VS Code, show _calculate_priority]**

> "Here's the priority engine — pure business logic."

```python
def _calculate_priority(damage_severity, damage_types, intent, urgency):
    if urgency == "high":                          return "high"
    if intent in HIGH_PRIORITY_INTENTS:            return "high"
    if damage_severity == "severe":                return "high"
    if any(d in HIGH_PRIORITY_DAMAGES for ...):    return "high"
    if urgency == "medium":                        return "medium"
    if intent in MEDIUM_PRIORITY_INTENTS:          return "medium"
    if damage_severity == "moderate":              return "medium"
    return "low"
```

> "Six rules, in order of severity.
> Emergency or insurance claim → always HIGH.
> Severe damage like shattered glass → always HIGH.
> Repair or warranty → MEDIUM.
> Everything else → LOW.
>
> Simple, auditable, configurable."

## SCENE 11 — Training the Local Models (6:45 – 7:30)

**[Screen: Show training script, don't actually run — just narrate]**

> "Right now the system is using Claude Vision as a fallback
> because we haven't trained the local models.
>
> With the Kaggle datasets downloaded,
> training is a single command:"

```bash
# Download datasets
python scripts/download_datasets.py --all

# Train vehicle type classifier — EfficientNet-B0 on 5600 images
python -m training.train_classifier --epochs 20 --batch-size 32

# Train damage detector on Car Damage Detection dataset
python -m training.train_damage_detector --stage 2 --epochs 20
```

> "The training script uses Albumentations for augmentation —
> flipping, brightness, blur, rotation —
> to make the model robust to real CCTV conditions.
>
> We use a two-phase approach:
> backbone frozen for 5 epochs to train the head,
> then unfrozen for fine-tuning.
> Early stopping prevents overfitting.
>
> Expected accuracy: 90–95% for vehicle classification,
> 85–90% for damage detection.
>
> Once trained, inference drops from 5 seconds
> to under 300 milliseconds — fully local, no API cost."
